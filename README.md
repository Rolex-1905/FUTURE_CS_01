# FUTURE Interns – Cyber Security Task 1

## Task Title
Vulnerability Assessment Report for a Live Website

## Objective
To perform a passive security assessment on a public website and identify common security weaknesses using ethical, non-intrusive methods.

## Target
Public demo website (public-facing pages only)

## Scope
- Public pages only
- Passive scanning and configuration analysis
- No exploitation, brute force, or DoS activities

## Tools Used
- OWASP ZAP (Passive Scan)
- Nmap
- Browser Developer Tools
- Canva (Report Design)

## Key Findings
- Lack of HTTPS encryption
- Missing HTTP security headers
- Exposure of HTTP service on port 80
- DOM-based Cross-Site Scripting
- Insufficient input validation and request protection
- Information disclosure via server headers

## Deliverables
- Vulnerability Assessment Report (PDF)
- Supporting evidence screenshots
- Tool output summaries

## Disclaimer
This assessment was conducted strictly for educational purposes as part of the Future Interns Cyber Security Internship. No exploitation or harmful activity was performed.
